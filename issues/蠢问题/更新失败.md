# 更新失败
## 原因
连不上git服务器，为什么连不上？因为人服务器压根就不在国内     
## 解决方案
为git配置代理    
以windows为例，需要在environments\MinGit\cmd目录打开cmd，输入
```cmd
git config --global http.proxy http://127.0.0.1:{http代理端口}`
git config --global https.proxy http://127.0.0.1:{http代理端口}
```
如
```cmd
git config --global http.proxy http://127.0.0.1:10809
git config --global https.proxy http://127.0.0.1:10809
```
## 衍生蠢问题
明明看到already up-to-date，怎么还是旧版本？     
部分更新源版本滞后，在这些更新源上的版本并不由Eridanus项目维护，与github.com上的版本同步情况取决于各更新源各自的维护者。
